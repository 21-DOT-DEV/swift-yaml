## 1. Goal & success criteria

Give the (later) surgical-edit layer a way to answer, from a parsed node: **"where in the original source text does this value sit?"** ‚Äî precisely enough to overwrite just that value and leave every comment, blank line, key order, and quote elsewhere byte-identical.

**Roadmap success metric (L51‚Äì53):** an authored `yamlx::` helper returns a node's mark, and spans are verified against **known fixture positions**.

**This plan's concrete criteria:**
- `yamlx::mark(node)` returns the node's start position (byte offset, line, column) for scalars, maps, and sequences; asserted equal to hand-verified positions in spec-derived fixtures.
- `yamlx::valueSpan(node, source, len)` returns the same start **plus** an end for a plain single-line scalar, with `endReliable == true`, such that `source[pos ..< end]` is exactly the scalar's bytes.
- For a quoted, block/folded/multi-line, or flow value ‚Äî **including empty quoted (`""` / `''`), empty block (`|`), and quoted scalars whose value begins with the wrapper quote (`''''` ‚Üí `'`, `"\""` ‚Üí `"`)** ‚Äî `valueSpan` returns `endReliable == false` (start still valid); the deferral is explicit and tested.
- An invalid/undefined node yields `ok == false`, never a crash (the guarded-value contract).

## 2. Scope

**In scope:** two authored C++ `inline` helpers and their POD return structs in `yamlcppShims`; a test suite in `Tests/yamlcppTests/` that pins positions and the reliable/deferred boundary.

**Out of scope (explicit owners):** any Swift public API / read-only position accessor ‚Üí **feature 1 or later**; the actual in-place `set` splice and `unset` ‚Üí **features 1‚Äì2** (they consume `valueSpan`); end-spans for quoted / multi-line / flow values ‚Üí **feature 1** (the "fiddly cases"); `CStr` non-escapable hardening ‚Üí **separate package-wide item behind PR #1's guard** (D5); Phase 4's public source-marks surface ‚Üí **Phase 4** (reuses `mark`).

## 3. Design ‚Äî the C++ bridge helpers

All additions go in `Sources/yamlcppShims/include/yamlcpp_shims.h`, under `namespace yamlx`, next to `parse` / `scalarText` / `mapValue`. Add `#include <cstring>` (for `std::memcmp`). Two value structs (plain integers ‚Äî D3) and two guarded helpers:

```cpp
// A node's start position. ok == false ‚áí invalid/undefined node (Node::Mark()
// throws InvalidNode on an invalid node ‚Äî guarded here ‚Äî and returns
// null_mark() for an undefined one). Offsets are 0-based, as yaml-cpp reports
// them; the (future) overlay adds 1 to line/column for human display.
struct Mark { bool ok; long pos; long line; long column; };

inline Mark mark(const Node &node) {
  Mark m{false, -1, -1, -1};
  try {
    YAML::Mark k = node.Mark();          // throws InvalidNode ‚Üí caught below
    if (k.is_null()) return m;           // undefined/empty node ‚áí ok stays false
    m = Mark{true, k.pos, k.line, k.column};
  } catch (...) {}
  return m;
}

// A scalar value's source span. `end`/`endReliable` are meaningful ONLY when
// endReliable == true: a plain, unquoted, single-line scalar whose SOURCE bytes
// equal its parsed bytes (verified ‚Äî not guessed, see D4). Any other shape, or a
// pos/byte skew from a BOM or CRLF input, leaves endReliable == false so the
// caller falls back to its own end-detection. `source`/`len` are the original
// document bytes (the caller already holds them); read-only, not retained.
struct Span { bool ok; long pos; long line; long column; long end; bool endReliable; };

inline Span valueSpan(const Node &node, const char *source, long len) {
  Mark m = mark(node);
  Span s{m.ok, m.pos, m.line, m.column, -1, false};
  if (!m.ok) return s;
  try {
    if (node.IsScalar() && source && m.pos >= 0) {
      const std::string &v = node.Scalar();
      long n = static_cast<long>(v.size());
      if (n > 0 &&                                                    // non-empty: memcmp(‚Ä¶,0) is vacuously equal
          m.pos + n <= len &&
          v.find('\n') == std::string::npos &&                       // single line
          source[m.pos] != '\'' && source[m.pos] != '"' &&           // not single/double quoted (marks at the quote)
          source[m.pos] != '|'  && source[m.pos] != '>'  &&          // not literal/folded block (marks at the indicator)
          std::memcmp(source + m.pos, v.data(), static_cast<size_t>(n)) == 0) {  // verbatim
        s.end = m.pos + n;
        s.endReliable = true;
      }
    }
  } catch (...) {}
  return s;
}
```

**Why this is correct (verified against the vendored source):**
- `Node::Mark()` exists and returns `YAML::Mark{pos, line, column}` (`node.h:52`, `impl.h:69‚Äì73`); `pos` is a real running **byte** offset ‚Äî `stream.cpp:298` increments `m_mark.pos` once per byte popped from the decoded-UTF-8 readahead (`std::deque<char>`, `stream.h:50`), not per Unicode character (`mark.h:20`). This byte granularity is what lets the Swift side index `Array(source.utf8)` by the same offsets. It throws `InvalidNode` on an invalid node and returns `null_mark()` (`-1,-1,-1`) on an undefined one ‚Äî both handled.
- **The discriminator is the first source byte, not an incidental `memcmp` mismatch.** Parse discards scalar style (D4), so we separate plain from non-plain *at the source*: a quoted or block scalar always **marks at its indicator** ‚Äî `ScanQuotedScalar`/`ScanBlockScalar` capture the mark before consuming `'` / `"` / `|` / `>` (`scantoken.cpp:355`, `:383`) ‚Äî and a plain scalar can **never begin** with any of those four (YAML 1.2 plain-scalar rule; `exp.h:151` `PlainScalar` regex excludes ``,[]{}#&*!|>'"%@` ``). So `source[m.pos] ‚àâ { ' " | > }` cleanly identifies a plain scalar; the `memcmp` then only verifies the span is byte-for-byte the parsed value. (Relying on `memcmp` alone is *not* safe: a quoted value that begins with its own wrapper quote ‚Äî `k: ''''` ‚Üí `'` ‚Äî matches the opening quote and would falsely pass. See round-2 log.) Flow collections are not scalars, so `IsScalar()` short-circuits them.
- **The `n > 0` guard is load-bearing.** `std::memcmp(p, q, 0)` is *vacuously* equal (returns 0 for a zero count ‚Äî [cppreference](https://en.cppreference.com/w/cpp/string/byte/memcmp)), so without it an empty quoted (`""` / `''`) or empty block (`|`) scalar ‚Äî where `Scalar()` is `""` and `n == 0` ‚Äî would falsely pass and report `endReliable == true` at a zero-width span (a later splice would then insert *before* the quote/indicator). Empty *plain* scalars cannot occur ‚Äî they parse as null, so `IsScalar()` is false ‚Äî so requiring `n > 0` defers exactly the empty quoted/block styles and nothing legitimate. This is the standard secure-coding rule: validate length before `memcmp`, never rely on a zero-length compare.
- `source`/`len` are needed (not derivable from the node) and are passed per call rather than retained in `ParseResult`: `mark(node)` stays source-free for Phase 4 reuse, and `parse()` isn't burdened with a document copy when spans aren't wanted. The BOM/CRLF pos-vs-source skew is neutralised by the same gate (mismatch ‚áí defer).

## 4. Tests

New `@Suite` in `Tests/yamlcppTests/yamlcppTests.swift` (imports `yamlcppShims`, `.Cxx` already on). Navigate to value nodes with the existing `yamlx.at` / `yamlx.atIndex` helpers, then assert:

1. **Known positions (the roadmap metric).** Reuse the spec-example mapping already in the file (`hr:  65 ‚Ä¶` / `avg: 0.278 ‚Ä¶` / `rbi: 147 ‚Ä¶`, yaml-cpp spec example 2.2). For each value assert `mark(node)` line/column against hand-verified values, and assert `valueSpan(...).endReliable`, then prove the span by decoding the spanned bytes: `let b = Array(source.utf8); #expect(String(decoding: b[pos ..< end], as: UTF8.self) == "65")` ‚Äî this pins the offsets empirically without hard-coding `pos`. Do **not** write `b[pos ..< end] == Array("65".utf8)`: subscripting `[UInt8]` yields an `ArraySlice<UInt8>`, and Swift 6 has no `==` between `ArraySlice<UInt8>` and `[UInt8]` ‚Äî decode to `String` as shown, or use `b[pos ..< end].elementsEqual("65".utf8)`.
2. **Deferred shapes ‚áí `endReliable == false`, `ok == true`.** Small authored fixtures: a double-quoted value (`k: "65"`), a single-quoted value, a literal/folded block scalar, a multi-line folded plain scalar, a flow collection value (`k: [1, 2]`); the zero-length-`memcmp` edge ‚Äî **empty double-quoted (`k: ""`), empty single-quoted (`k: ''`), empty block (`k: |`)**; and the **wrapper-quote-prefix edge ‚Äî `k: ''''` (value `'`), `k: ''''''` (value `''`), `k: "\""` (value `"`)**, where the parsed value begins with the same quote the wrapper uses. Each asserts the start is still returned (`ok == true`) and the end is withheld (`endReliable == false`).
3. **Guarded failure.** An undefined node (e.g. a missing key via `yamlx.at`) ‚áí `ok == false`, no crash.
4. **Sequence & map starts.** `mark` on the root sequence (spec example 2.1) and on a nested map returns sensible line/column (start-only; `endReliable == false`), covering the "any node" position claim and the Phase-4-reuse primitive.

Fixtures are spec-derived or minimal authored strings with positions computed by hand and pinned by the slice check (A4).

## 5. Risks & mitigations

- **Zero-length `memcmp` (empty scalars).** `std::memcmp(‚Ä¶, 0)` returns 0 (vacuously equal), which would mark an empty quoted (`""` / `''`) or block (`|`) scalar reliable at a zero-width span. *Mitigation:* the `n > 0` guard in the gate (¬ß3, D4); empty double-/single-quoted and empty block fixtures assert `endReliable == false` (¬ß4). Empty *plain* scalars can't occur ‚Äî they parse as null (`IsScalar()` false).
- **Wrapper-quote-prefix false positive.** A quoted scalar whose *value starts with the wrapper quote* (`k: ''''` ‚Üí `'`; `k: "\""` ‚Üí `"`) has `source[pos]` equal to the opening quote *and* to the value's first byte, so `memcmp` alone matches a truncated span. *Mitigation:* the explicit `source[pos] ‚àâ { ' " | > }` guard (¬ß3, D4) ‚Äî a plain scalar can never start with those (YAML 1.2 / `exp.h:151`); fixtures pin the deferral (¬ß4).
- **Alias nodes report the *anchor's* position.** `NodeBuilder::OnAlias` ignores the alias token's mark and pushes the anchor's underlying node (`nodebuilder.cpp:37-41`), so `mark`/`valueSpan` on an alias (`*a`) return the anchor value's source position, not the alias token's. Pre-existing yaml-cpp behavior, not a defect here; **feature 1 owns the decision** whether editing an alias means editing the shared anchor value or replacing the `*a` token. Flagged so it isn't a surprise.
- **`pos` vs. original bytes ‚Äî a BOM skews, CRLF does not.** yaml-cpp's `pos` counts *raw* UTF-8 bytes: `StreamInUtf8` pushes each byte without stripping `\r` (`stream.cpp:332-337`), so **CRLF does not skew** ‚Äî a plain value's `pos` equals its offset in the original text and it spans reliably (verified ‚Äî `handlesLineEndingsAndBOM`). A **leading BOM** *is* the skew case: it is consumed during charset detection *before* `pos` counting starts (`stream.cpp:245`), so `pos` is post-BOM while our source bytes include it. *Mitigation:* the `memcmp` gate makes that skew fail closed ‚Äî `endReliable == false`, start still reported, never a mis-splice (verified ‚Äî same test). Feature 1 owns exact handling of BOM'd inputs. (Round-1's ¬ß5 wording had this backwards ‚Äî it named CRLF as the degradation; corrected here.)
- **Block-scalar mark position.** If a `|`/`>` block scalar's mark lands on content rather than the indicator, the folded/literal newlines and chomping still break the byte match (and `v.find('\n')` guard). *Mitigation:* covered by a block-scalar fixture asserting deferral.
- **Interop regression.** *Mitigation:* only integer-returning value helpers are added ‚Äî no `std::string` crosses, no `CStr` involved ‚Äî so PR #1's `-c release` whole-module guard is unaffected; CI runs it regardless.
