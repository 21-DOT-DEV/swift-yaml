## 1. Goal & success criteria

Let a caller give a path a scalar value that **isn't there yet** ŌĆö filling a null, or adding a new key line ŌĆö with the only textual change being the filled span or the one added line; everything else byte-identical; the result re-parses to the intended value. Add-only: a path that already holds a real value returns a typed error, never a silent overwrite.

**Concrete criteria:**
- `insert("a:", at: ["a"], to: 1)` ŌåÆ `a: 1`; likewise `a: ~` and `a: null` ŌåÆ `a: 1`. Re-parses to `1`.
- `insert(block, at: ["server", "tls"], to: true)` where `server` is an existing block map without `tls` ŌåÆ a new `tls: true` line appended after the last `server` child, at the children's indentation; siblings and comments byte-identical; re-parses.
- Target already holds a real value ŌåÆ `valueAlreadyPresent` (use `set` to change it).
- Target is an empty string (`""`/`''`) ŌåÆ `valueAlreadyPresent` (it's a real value).
- Parent block is missing, is a list, or is written inline (`{ŌĆ”}`) ŌåÆ typed error, never a corrupt edit.
- A reference-using or malformed document ŌåÆ typed error.

## 2. Scope

**In scope:** `insert` (static); **fill** a null scalar leaf; **insert** a new leaf key into an existing **block** map (append at end, sibling indentation); reuse of the navigation, guards, value/error/serializer machinery; tests.

**Out of scope (typed errors / deferred owners):** **creating missing parent blocks** (`["a","b","c"]` where `a.b` is absent) ŌĆö a later increment (nested multi-line insertion); **inserting into an inline `{ŌĆ”}` map** or **appending to a list** ŌåÆ `unsupportedValueShape` (different mechanics; deferred); **overwriting a real value** ŌåÆ `valueAlreadyPresent` (that is `set`'s job); **positional / sorted insertion** and **attaching a comment to the new key** (ruamel offers these; a later opt-in); **multi-document** input ŌåÆ first document only.

## 3. Design / approach

**Public surface** (added to the existing `YAMLEditor`):
```swift
extension YAMLEditor {
    /// Writes `value` at `path` when it isn't there yet: fills a null value, or
    /// adds a new key (appended at the end of its existing block, at the siblings'
    /// indentation). Add-only ŌĆö throws `valueAlreadyPresent` if a real value is
    /// already there (use `set` to change it). Everything else stays byte-identical.
    public static func insert(_ yaml: String, at path: [YAMLPath.Component], to value: Value) throws -> String
}
```

**Pipeline** (all Swift over the source bytes; reuses `set`/`unset`'s parse + guarded navigation):
1. **Guarded parse; refuse anchor/alias** documents (the existing scan).
2. **Navigate `path`, validating each step.** Split on the last component:
   - **Full path resolves to a node** ŌåÆ it exists; classify:
     - **null node** ŌåÆ **fill** (step 3a). Detection is by the parser's **node kind** (D5): a value the parser labels *null* is fillable; `""`/`''` are labeled *scalar* (a real value) and fall to the next case. (A probe confirmed 0.6.2 labels the no-value spellings null and `""` scalar; the source-byte check the first draft described was dropped ŌĆö see ┬¦7.)
     - **real scalar** ŌåÆ `valueAlreadyPresent`.
     - **map / sequence** ŌåÆ `notASingleValue`.
   - **Last component absent, but its parent resolves** ŌåÆ candidate to *insert* (step 3b). Parent must be a **block map**: an inline `{ŌĆ”}` map or a sequence/non-map parent ŌåÆ `unsupportedValueShape`; a parent that doesn't resolve at all ŌåÆ `pathNotFound` (no `mkdir -p`).
3. **Edit:**
   - **(3a) Fill.** The null's node mark is unreliable (for a bare `key:` it lands on the *next* line), so scan back from it to the key's line, find the key/value colon (`keyValueColon`, quote-aware), and rewrite the value slot: `<indent>key: <token>`, dropping whatever null spelling sat there and **keeping any same-line trailing comment plus the whitespace before it** (like `set`). `token = emitScalarToken(value)`.
   - **(3b) Insert.** The parent (block) map's mark points at its **first child key** ŌåÆ the children's indentation. Walk the block to the **dedent** (indentation drops below the children) and append `<indent><key>: <token>` after the block's last **data** line ŌĆö before a trailing block-level comment/blank ŌĆö matching the document's line ending. A flow (`{ŌĆ”}`) parent is caught by a byte check (`{` at the parent's mark). The **key** is emitted quoted if it needs quoting.
4. **Return** `String(decoding: splicedBytes, as: UTF8.self)`.

**New/notable helpers:** the key-line rebuild for fill (scan back to the key's line, preserve the trailing comment + its whitespace); the **dedent-scan block-end finder** (the core new mechanic); `keyValueColon` (quote-aware); a byte check for a flow parent (`{` at the parent's mark); the key/value `emitScalarToken` (quote-if-needed).

## 4. Tests (`Tests/YAMLTests/InsertTests.swift`)

- **Fill:** bare `a:`, `a: ~`, `a: null` ŌåÆ `a: <value>`; number vs string intent (`to: 1` plain, `to: "1"` quoted); oracle re-parse.
- **Insert new key:** append at end of a block, sibling indentation, siblings + comments byte-identical (Example 2.2-style commented fixture); nested block; a key that needs quoting is emitted quoted.
- **Refusals (typed error, never corruption):** real value present; empty string `""`/`''` present; inline `{ŌĆ”}` parent; list/sequence parent; missing parent (no `mkdir -p`); reference-using doc; malformed doc.
- **Preservation:** comments and blank lines around the edit unchanged; a filled value keeps a same-line trailing comment; a multi-line document touches only the target line.
- **Bump tripwires (`EngineNullContract`, a separate suite):** pin the engine behavior `insert` (and decoding) rely on ŌĆö the no-value spellings label as *null*, `""`/`''` label as *scalar*, and a no-value decodes to `nil` (not `"null"`). A future yaml-cpp update that shifts any of these fails loudly and points at the cause ([#1290](https://github.com/jbeder/yaml-cpp/issues/1290)-class regressions).

## 5. Risks & mitigations

- **A future engine bump changing null handling.** *Not* the current engine ŌĆö a probe confirms 0.6.2 labels no-value *null* and `""`/`''` *scalar*. The cited [yaml-cpp #1290](https://github.com/jbeder/yaml-cpp/issues/1290) is a **0.8.0** *read-null-as-text* bug on the **decode** path, not a classification bug here. *Mitigation:* **bump-tripwire tests** (`EngineNullContract`, ┬¦4) pin the classification (`a:`/`~`/`null` ŌåÆ null; `""`/`''` ŌåÆ scalar) and that a no-value decodes to `nil` (not `"null"`); a bump that shifts either fails loudly and names the cause.
- **Parent-map extent detection (the core new mechanic).** Finding "the last child's last line" for the append point is where correctness lives ŌĆö nested blocks, a last child with a multi-line value, trailing comments/blank lines. *Mitigation:* spec-derived fixtures + the re-parse oracle; when in doubt, refuse rather than mis-place.
- **Indentation matching.** *Mitigation:* copy a sibling key's exact leading whitespace; never assume a 2-space step.
- **Inline `{ŌĆ”}` / list parents.** *Mitigation:* refuse (as `set`/`unset` refuse inline collections) ŌĆö never a partial edit.
- **Key needing quotes.** *Mitigation:* emit the inserted key through the same needs-quoting logic as scalar values.
